Variables Info

Variables are like a map for your commands. They tell the commands how to get certain information.

Let's say you wanted a command to show the current game. You could manually update the command every stream. Instead you can use a variable.

Variables will search for the data that they are supposed to hold and insert themselves into the command. A complete list of them is found below.

  • $target : The first word after the command. ex. !so Mytho : Go follow Mytho at glimesh.tv/Mytho
  • $user : The user who said the command
  • $watchtime : The amount of watchtime of the user in minutes
  • $cmdcount : The number of times the command has been used.
  • $advice : Returns random advice.
  • $discord : Returns the channel's discord URL.
  • $guilded : Returns the channel's guilded URL.
  • $instagram : Returns the channel's instagram URL.
  • $youtube : Returns the channel's youtube channel URL.
  • $twitter : Returns the channel's twitter URL.
  • $catfact : Returns a random cat fact. (May exceed message length)
  • $dogfact : Returns a random dog fact. (May exceed message length)

System Commands

GlimBoi has built in commands (system commands). These commands cannot be deleted or modified.

You should avoid having commands with conflicting names. A system command will take priority.

System commands help the bot and inform your viewers about its capabilities. A complete list of them is found below.

  • !commands : A list of commands
  • !quote : Sends a random quote.
  • !user (add/new, remove/del/delete) : User functions. Ex '!user new Mytho' would add Mytho to the user list.
  • !quote (add/new) USER QUOTE : Adds a new quote. Ex. !quote add Mytho Hello world!
  • !points : Returns how many points you have.
  • !points top : Returns the user with the most points.
  • !points NUMBER : Returns the points of the user with the position you typed.
  • !points (add/inc/+) USER AMOUNT : Increases a users point value
  • !points (sub/dec/-) USER AMOUNT : Lowers a users point value
  • !points (set/=) USER COUNT : Sets a users point value
  • !points get USER : Returns a users point value
  • !points help : Displays the above syntax
  • !rank : Returns the rank of the user.
